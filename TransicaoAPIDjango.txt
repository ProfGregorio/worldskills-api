Sim. É possível usar o Swagger com o Django REST Framework (DRF) para documentar e testar sua API de forma interativa.

Atualmente, a opção mais recomendada é o drf-spectacular, que gera uma especificação OpenAPI 3 e fornece interfaces do Swagger UI e ReDoc.

Instalação
pip install drf-spectacular

Adicione ao INSTALLED_APPS:

INSTALLED_APPS = [
    ...
    "drf_spectacular",
]

Configure o DRF em settings.py:

REST_FRAMEWORK = {
    "DEFAULT_SCHEMA_CLASS": "drf_spectacular.openapi.AutoSchema",
}

Opcionalmente, configure o Spectacular:

SPECTACULAR_SETTINGS = {
    "TITLE": "Minha API",
    "DESCRIPTION": "Documentação da API",
    "VERSION": "1.0.0",
}
URLs

No urls.py:

from django.urls import path
from drf_spectacular.views import (
    SpectacularAPIView,
    SpectacularSwaggerView,
    SpectacularRedocView,
)

urlpatterns = [
    path("api/schema/", SpectacularAPIView.as_view(), name="schema"),
    path(
        "api/docs/",
        SpectacularSwaggerView.as_view(url_name="schema"),
        name="swagger-ui",
    ),
    path(
        "api/redoc/",
        SpectacularRedocView.as_view(url_name="schema"),
        name="redoc",
    ),
]

Depois, acesse:

http://localhost:8000/api/docs/ → Swagger UI
http://localhost:8000/api/redoc/ → ReDoc
http://localhost:8000/api/schema/ → Especificação OpenAPI
Documentando endpoints

Você pode enriquecer a documentação com decorators:

from drf_spectacular.utils import extend_schema

@extend_schema(
    summary="Lista usuários",
    description="Retorna todos os usuários cadastrados."
)
class UserListView(ListAPIView):
    queryset = User.objects.all()
    serializer_class = UserSerializer
